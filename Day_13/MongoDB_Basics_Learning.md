# Day 12 - Learning MongoDB Basics

## What I worked on

### Work Summary
Today I started learning about MongoDB and database fundamentals. I understood what a database is and how applications use databases to store and retrieve information such as user data, products, and transactions.

I studied the difference between SQL and NoSQL databases. SQL databases store data in tables with rows and columns, while NoSQL databases store data in flexible formats such as documents, key-value pairs, or graphs.

I also learned that MongoDB is a NoSQL document database that stores data in JSON-like documents. Instead of tables, MongoDB uses collections and documents. This makes it easier for developers to manage complex and changing data structures.

I explored the structure of MongoDB which follows the hierarchy:
Database → Collections → Documents → Fields.
