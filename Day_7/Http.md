# Day 7-- Understanding REST API Concepts

## Work Summary

Today I focused on learning the fundamentals of REST APIs
(Representational State Transfer) and how they are used in modern web
applications. I studied how APIs enable communication between the
frontend and backend systems by sending and receiving data through HTTP
requests.

I explored the main HTTP methods used in REST APIs such as GET, POST,
PUT, and DELETE. These methods allow applications to retrieve, create,
update, and delete data on the server. I also learned about common HTTP
status codes like 200 (success), 201 (created), 404 (not found), and 500
(server error).

Additionally, I studied the structure of an API request including URL,
headers, request body, and query parameters. I reviewed examples of API
endpoints and documented the concepts learned for future backend
development practice.
